# Plan : Implémentation e-commerce complet — Green Mood CBD

## Contexte
La boutique passe d'un site vitrine statique à une plateforme e-commerce complète avec catalogue dynamique (Supabase), panier, paiement Viva Wallet, comptes utilisateurs, et gestion des stocks. L'utilisateur a déjà un projet Supabase actif, pas encore de compte Viva Wallet (placeholders), et souhaite un admin panel intégré — tout en une fois.

---

## Dépendances à installer

```bash
npm install @supabase/supabase-js zustand
```

## 1. Base de données Supabase

### Script SQL à exécuter dans Supabase Dashboard → SQL Editor

**Tables :**

```sql
-- Catégories
CREATE TABLE categories (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  slug text UNIQUE NOT NULL,
  name text NOT NULL,
  description text,
  icon_name text,
  image_url text,
  sort_order int NOT NULL DEFAULT 0,
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Produits
CREATE TABLE products (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  category_id uuid NOT NULL REFERENCES categories(id),
  slug text UNIQUE NOT NULL,
  name text NOT NULL,
  description text,
  cbd_percentage numeric(5,2),
  thc_max numeric(5,3),
  weight_grams numeric(8,2),
  price numeric(10,2) NOT NULL,
  image_url text,
  stock_quantity int NOT NULL DEFAULT 0,
  is_available boolean NOT NULL DEFAULT true,
  is_featured boolean NOT NULL DEFAULT false,
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Profils utilisateurs (étend auth.users)
CREATE TABLE profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  full_name text,
  phone text,
  loyalty_points int NOT NULL DEFAULT 0,
  is_admin boolean NOT NULL DEFAULT false,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Adresses de livraison
CREATE TABLE addresses (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  label text NOT NULL DEFAULT 'Domicile',
  street text NOT NULL,
  city text NOT NULL,
  postal_code text NOT NULL,
  country text NOT NULL DEFAULT 'France',
  is_default boolean NOT NULL DEFAULT false,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Commandes
CREATE TABLE orders (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES profiles(id),
  status text NOT NULL DEFAULT 'pending',
  -- status: pending | paid | processing | ready | shipped | delivered | cancelled
  delivery_type text NOT NULL DEFAULT 'click_collect',
  -- delivery_type: click_collect | delivery
  address_id uuid REFERENCES addresses(id),
  subtotal numeric(10,2) NOT NULL,
  delivery_fee numeric(10,2) NOT NULL DEFAULT 0,
  total numeric(10,2) NOT NULL,
  loyalty_points_earned int NOT NULL DEFAULT 0,
  viva_order_code text,
  payment_status text NOT NULL DEFAULT 'pending',
  notes text,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Lignes de commande
CREATE TABLE order_items (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id uuid NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id uuid NOT NULL REFERENCES products(id),
  product_name text NOT NULL,
  unit_price numeric(10,2) NOT NULL,
  quantity int NOT NULL,
  total_price numeric(10,2) NOT NULL
);

-- Mouvements de stock
CREATE TABLE stock_movements (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id uuid NOT NULL REFERENCES products(id),
  quantity_change int NOT NULL,
  type text NOT NULL, -- sale | restock | adjustment | return
  note text,
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Trigger: créer un profil auto à l'inscription
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, full_name)
  VALUES (new.id, new.raw_user_meta_data->>'full_name');
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
```

**RLS Policies :**
- `categories` / `products` : lecture publique, écriture admin uniquement
- `profiles` : lecture/écriture par l'utilisateur lui-même ou admin
- `addresses` : lecture/écriture par propriétaire
- `orders` / `order_items` : lecture par propriétaire ou admin, création authentifiée
- `stock_movements` : admin uniquement

Migrer les 12 produits hardcodés de `Products.tsx` vers la DB avec leurs catégories.

## 5. Architecture des stores Zustand

### `cartStore.ts`
```typescript
interface CartItem {
  product: Product;
  quantity: number;
}
interface CartStore {
  items: CartItem[];
  isOpen: boolean;           // sidebar ouverte
  deliveryType: 'click_collect' | 'delivery';
  addItem: (product: Product) => void;
  removeItem: (productId: string) => void;
  updateQuantity: (productId: string, qty: number) => void;
  clearCart: () => void;
  toggleSidebar: () => void;
  setDeliveryType: (type: ...) => void;
  // computed
  total: number;
  itemCount: number;
}
// Persisté via zustand/middleware persist → localStorage
```

### `authStore.ts`
```typescript
interface AuthStore {
```

## 6. Flux Viva Wallet (placeholders prêts pour activation)

L'intégration utilise l'Express server déjà installé (`server/index.ts`) :

```
Client → POST /api/payment/create-order
  → Express → Viva Wallet API (créer order)
  → Retourne orderCode
  → Redirect vers Viva Wallet checkout URL

Viva Wallet → Webhook POST /api/payment/webhook
  → Vérifier signature
  → Mettre à jour order.payment_status = 'paid'
  → Décrémenter stock
  → Ajouter loyalty_points
  → Redirect vers /commande/confirmation
```

Fichiers serveur : `server/index.ts`, `server/routes/payment.ts`, `server/routes/orders.ts`

En mode placeholder (sans compte Viva Wallet) : simuler le paiement avec un bouton "Payer (simulation)" qui marque l'ordre comme `paid`.

## 7. Admin Panel (`/admin`)

Protégé par `is_admin = true` dans `profiles`.

Sections :
- **Produits** : tableau CRUD (créer, éditer prix/stock/disponibilité, désactiver)
- **Commandes** : liste avec statuts, mise à jour statut (processing → ready → shipped)
- **Stocks** : ajustement manuel des quantités + historique des mouvements
- **Fidélité** : consultation des points par utilisateur

---

## 8. Fidélité (Loyalty)

Règle simple : **1€ dépensé = 1 point**. 100 points = -5€ de réduction.
- Points calculés et enregistrés à la confirmation du paiement
- Affichés dans `/compte` (tableau de bord)
- Déductibles au checkout (champ optionnel "Utiliser mes points")
